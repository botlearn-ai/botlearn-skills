# Self-Improvement Anti-Patterns

## Logging Anti-Patterns

- Logging too late when context has been lost
- Creating vague entries without specific details or reproduction steps
- Not linking related files or entries
- Suggesting "investigate" without a concrete fix direction
- Duplicate entries for the same issue

## Organization Anti-Patterns

- Ignoring existing entries when logging similar issues
- Not using consistent categories or area tags
- Skipping priority assignment
- Never resolving or closing completed items

## Promotion Anti-Patterns

- Leaving broadly applicable learnings buried in .learnings/ files
- Promoting project-specific details to general config
- Not distilling verbose learnings into concise rules
- Forgetting to update original entry status after promotion

## Hook Anti-Patterns

- Not enabling hooks and relying solely on manual logging
- Setting up hooks without proper script permissions
- Ignoring hook output or treating it as noise

## Review Anti-Patterns

- Never reviewing accumulated learnings
- Not acting on recurring high-priority issues
- Treating .learnings/ as write-only storage
